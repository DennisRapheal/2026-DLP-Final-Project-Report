	\begin{block}{Project Overview}
		
		OpenPI-Comet predicts long-horizon 32-step action chunks for BEHAVIOR-1K,
		but a fixed chunk can become locally stale as the robot interacts with the
		scene. We study whether an Asynchronous Action Chunk Correction (A2C2)
		module can improve a frozen OpenPI-Comet VLA policy without retraining the
		base model.
		
		At execution offset $k$, A2C2 uses the latest observation to predict a
		23-D residual on top of the selected base action:
		$$
		a^{exec}_{t+k} = a_{t,k} + \lambda \Delta a_{t,k}.
		$$
		We first reproduce fixed-seed OpenPI-Comet rollouts, then evaluate whether
		raw and normalized residual objectives can reduce action-level error without
		hurting closed-loop task performance.

		
	\end{block}
	\end{column}

			\begin{alertblock}{Why Did A2C2 Hurt Performance?}
				
				Even though the validation corrected MSE is roughly half of the base action MSE, 
				the evaluation performance still dropped significantly.
				Our diagnosis is that the current objective treats
				a heterogeneous, bounded robot action vector as a homogeneous residual
				regression problem:
				$$
				L = \|\hat{\Delta a}_{t,k} - \Delta a^*_{t,k}\|_2^2, 
				\quad
				\Delta a^*_{t,k} = a^*_{t+k} - a_{t,k}.
				$$
				
				\begin{itemize}
					\item \textbf{Scale mismatch:} arm/torso dimensions can dominate the loss,
					while small-range gripper errors remain weakly supervised.
					\item \textbf{Behavior mismatch:} gripper commands are event-like; MSE can
					average open/close actions into invalid half-open behavior.
					\item \textbf{Policy mismatch:} a large residual can perturb a coherent
					action chunk generated by the frozen flow policy.
				\end{itemize}
				
			\end{alertblock}

			\begin{defbox}{Range-Normalized Residual Loss}

				For each action dimension $j$, we estimate a robust scale from the training
				split and normalize residual errors:
				$$
				s_j = \max\left(Q_{0.99}(a_j)-Q_{0.01}(a_j), \epsilon\right),
				\quad
				e_j = \frac{\hat{\Delta a}_{t,k,j} - \Delta a^*_{t,k,j}}{s_j}.
				$$
				Then we aggregate errors by action group:
				$$
				L_{\text{group}} =
				\sum_{g \in \mathcal{G}} w_g
				\frac{1}{|g|}\sum_{j \in g} \rho(e_j),
				$$
				where $\mathcal{G}$ contains base, torso, left/right arm, and left/right
				gripper groups. We compare normalized MSE, normalized Huber, and a
				gripper-weighted Huber variant with $w_g=2.0$ for grippers and $w_g=1.0$
				otherwise. This directly tests whether the failure comes from scale
				imbalance, outlier residuals, or weak gripper supervision.

			\end{defbox}

			% \begin{thm}{Potential Risk: Random Chunk Supervision Can Break Mode Consistency}{}
				
			% 	The base flow policy samples a temporally coherent action chunk:
			% 	$$
			% 	A^b_t \sim \pi_b(A_t \mid o_t, \ell).
			% 	$$
			% 	However, A2C2 is trained with randomly sampled chunk offsets. For an offset
			% 	$k$, the residual target is defined locally as
			% 	$$
			% 	\Delta a^{target}_{t,k}
			% 	=
			% 	a^{expert}_{t+k} - a^b_{t,k}.
			% 	$$

			% 	This provides only local one-step supervision. If multiple valid chunks exist
			% 	under similar observations, the sampled expert action may belong to a different
			% 	mode from the base action chunk. Then the residual head is trained to partially
			% 	pull a coherent base chunk toward a locally different expert mode:
			% 	$$
			% 	a^b_{t,k} + \Delta a_\phi
			% 	\approx
			% 	a^{expert}_{t+k}.
			% 	$$

			% 	As a result, the correction may not collapse an entire trajectory, but it can
			% 	break chunk-level mode consistency. The corrected action sequence may mix
			% 	parts of different modes, reducing the temporal coherence originally produced
			% 	by the flow-based base policy.
				
			% \end{thm}

			\begin{block}{Discussion}
				
				The result is negative but informative: although our implementation follows
				the A2C2 setup with a frozen OpenPI-Comet and online residual correction,
				it exposes a clear offline-online gap. 
				
				\medskip

				A2C2 reduces action-level validation error, but degrades closed-loop task
				performance, suggesting that residual imitation accuracy alone is not
				sufficient for long-horizon robot control.
				
				\medskip
				\begin{itemize}
					\setlength{\itemsep}{0.6em}
					\item \textbf{Offline-online mismatch:} training states come from expert
					demonstrations, but online rollouts visit policy-induced states with missed
					grasps, contact errors, and shifted object poses.

					\item \textbf{Task-specialization mismatch:} OpenPI-Comet is trained as a
					multi-task policy, so its action chunks reflect a broad task-agnostic control
					prior. In contrast, the correction head is trained to fit expert-minus-base
					residuals from a selected task, which can force the current action toward a
					narrow task-specific distribution and disturb the base policy's general
					multi-task behavior.

					\item \textbf{Multimodal policy collapse:} the flow-matching action expert can
					sample diverse plausible action chunks under the same observation. A
					deterministic residual head trained with pointwise MSE may over-correct each
					sampled base action toward a single demonstrated expert action, reducing the
					multimodal behavior of the original flow policy.
				\end{itemize}
				
			\end{block}

			
			\begin{block}{Conclusion and Next Steps}
				A2C2 gives a negative but specific result: lower validation MSE does not
				predict online success. We propose three follow-up experiments that directly
				test the discussion hypotheses.
				
				\begin{itemize}
					\item Replay failed rollout states and
					sweep residual scaling, group-wise clipping, and action-range projection to
					test whether a gated Norm-MSE corrector can reduce Res. L2, OOR, gripper
					delay, and jerk while improving $q_{final}$.
			
					\item Train a broad mixed-task head
					on all available correction data, then compare it with target-task
					specialist heads and task-conditioned residual MLPs to separate
					over-specialization from cross-task negative transfer.
			
					\item Add chunk-level smoothness/jerk
					regularization and test multi-sample base-chunk selection to preserve the
					frozen flow policy's temporally coherent action modes.
				\end{itemize}
			\end{block}
